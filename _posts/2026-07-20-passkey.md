There is an open source alternative to commercial hardware keys like Yubikey. The alternative is COTS so it is not a secure element, and in my setup by default there is no secure boot and the efuses are not blown, so the key can be copied. Although it is possible with rp2350 and esp32-s3 to make it relatively hardened [1](https://github.com/TheMaxMur/RS-Key/#limitations-short-list) [2](https://github.com/librekeys/pico-fido2#security) (not possible with rp2040). But a device like this still has its applications, taking into account the threat model. The device uses FIDO2 authentication standard and works on both Windows and Linux. The following is how I set it up. 

Updates: 
twitter and google in Chrome and Firefox on Ubuntu are not accepting my s3 flashed with pico-fido, despite `fido2-token -I -d /dev/hidraw1` reporting `version strings: U2F_V2, FIDO_2_0, FIDO_2_1, FIDO_2_2`, even despite changing the VID:PID to one of a yubikey (1050:0406). s3 with https://github.com/librekeys/pico-fido2 has the same behavior. rp2350 as 1050:0407 with RS-Key firmware is not accepted as well, `U2F_V2, FIDO_2_0, FIDO_2_1, FIDO_2_2, FIDO_2_3`.

## hardware

I'm keeping it very minimalistic: RP2350 and ESP32-S3. For S3 I went with `Muse Lab ESP32-S3 USB Dongle`, which is very much like Lilygo t-dongle S3 but: without the screen (not needed), the sd card slot is inside the case (difficult to access, but I do not need the storage), the boot button is also difficult to access (not critical, I only need to press it once). and it's almost half the price, 51元. RP2350 is a better choice because it supports OTP fuses and secure boot, I went with the [深圳市云造科技网络 RP2350 USB](https://www.nologo.tech/product/raspberrypie/RP2350/rp2350usb/rp2350_usb1.html) (the LED driver is WS2812 Neopixel, GPIO#22), a tiny naked board with usb A - 26元. needs a heatshrink put onto it.

## firmware, rp2350

for rp2350 I would go with https://github.com/TheMaxMur/RS-Key/ as it supports all the features of the pico-fido2 (including post-quantum cryptography) and it was build specifically as an open-source alternative to the pico-fido2 that went commercial and nuked the open-source repo. for the record, there is also https://github.com/librekeys/pico-fido2 which is an attempt to save the last known open-source version of pico-fido2. 

for rp2350, on the linux host:

```bash
mkdir workdir

cat > Dockerfile <<'EOF'
FROM nixos/nix

RUN mkdir -p /root/.config/nix && echo "experimental-features = nix-command flakes"  > /root/.config/nix/nix.conf

RUN nix shell nixpkgs#git

WORKDIR /workdir
EOF

docker build -t rs-key-builder .

docker run -it --rm --device /dev/bus/usb -v workdir:/workdir -v nix:/nix -v home:/root rs-key-builder


```

then inside the container:

```bash

git --version
git clone https://github.com/themaxmur/RS-Key.git

#start dev console
cd RS-Key
nix develop

env VIDPID=Yubikey5 cargo build --release -p firmware
picotool uf2 convert target/thumbv8m.main-none-eabihf/release/firmware -t elf firmware.uf2
#connect the board in the bootloader mode
picotool info
picotool load -v firmware.uf2 && picotool reboot
```

restart the container (because the key was reconnected), start the dev console and `rsk fido set-pin`, or set it with picoforge

## firmware, s3

for my s3 I went with [the original pico fido](https://github.com/polhenarejos/pico-fido) firmware, flashed it in Chrome at https://www.picokeys.com/esp32-flasher/

must change the default VID=2E8A PID=10FE (easiest way to do it is with picoforge. or modify CMakeLists.txt and recompile pico-fido2) to make it look like a Yubikey.

the default pico-fido VID:PID key works in Chrome, and to make it work in Firefox (installed form snap) i had to:

```bash
lsusb | grep Pico
# gave me:
# Bus 001 Device 024: ID 2e8a:10fe Pol Henarejos Pico Key

sudo tee -a /etc/udev/rules.d/70-snap.firefox.rules <<- 'EOF'
SUBSYSTEM=="hidraw", KERNEL=="hidraw*", ATTRS{idVendor}=="2e8a", ATTRS{idProduct}=="10fe", TAG+="snap_firefox_firefox", TAG+="uaccess"
EOF

sudo udevadm control --reload-rules && sudo udevadm trigger
```

## management

I use [picoforge](https://github.com/librekeys/picoforge) to manage the device. Works on Windows and Linux, for Ubuntu I did

```bash
sudo apt install pcscd
sudo systemctl enable --now pcscd
wget https://github.com/librekeys/picoforge/releases/download/v0.7.1/picoforge_0.7.1_amd64.deb
sudo apt install ./picoforge_0.7.1_amd64.deb
picoforge
```

## initial setup

open `picoforge` and set the PIN. optionally, one can change the default lentgh of the PIN from 4 to something longer. picoforge listed secure boot as a feature for future releases, so it will be possible to make this storage more secure later.

## storing ssh keys

```bash
ssh-keygen -t ed25519-sk -f ~/.ssh/id_ed25519_sk    # touch twice, enter PIN
ssh-copy-id -i ~/.ssh/id_ed25519_sk you@server
ssh -i ~/.ssh/id_ed25519_sk you@server              # one touch to log in
```

## optional

might as well install `sudo apt install libfido2-1 libfido2-dev libfido2-doc fido2-tools` for additional diagnostics:

```bash
fido2-token -L
fido2-token -I  /dev/hidraw1
fido2-token -I -d /dev/hidraw1
```
